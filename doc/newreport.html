Creating a new report

If there is a tool that's not included here that you want to run, just provide a build script in the right place, and the framework will pick it up. A template for this is provided in the template folder. The steps below point out the places where you can put in pieces specific to the tool.
  • Create a directory for the report under report and put a build.xml file there. If you like, there is a template folder that you can use as a starting point with a commented build.xml. 
    cp -r template report/myreport
    Hmm, except that the .svn folder also comes along...
  • Create a one-line description. This will show up in the home page of the report package.
  • Have a report task that sandwiches your tool's task between the tasks init and transform, both of which are defined by the framework.
  • Include the reports-common.xml file. This has the property definitions, tasks and macrodefs that are available.
  • If the tool has an Ant task, define the taskdef
  • Right the task to run the report. Here you have available to you a number of useful properties.
    • ${src.dir}
    • ${bin.dir}
    • ${gen.report.dir}
